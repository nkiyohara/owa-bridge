Sixty-second setup

Install. Connect. Ask.

One local binary, one client command, then ordinary language.

  1. 01

    Install and sign in

    brew install nkiyohara/owa-bridge/owa-bridge
owa config init
owa login

    Windows and direct-download options are in the install guide.

  2. 02

    Connect your agent

    owa mcp setup codex
# or: claude-code, github-copilot,
# gemini-cli, qwen-code, qoder

    The command records absolute paths and tells you how to verify the connection.

  3. 03

    Start a new session

    Check Outlook and show what needs attention.

    Server instructions and the optional Agent Skill make the right tools discoverable.

Safety is the architecture

The agent never owns your session.

A local daemon owns the interactive browser session. Agents receive narrow tools, not credentials.

AgentstdioMCPlocal IPCSession ownerHTTPSOutlook Web
  • Metadata first. Sensitive bodies and attachments require explicit tools.
  • Preview, then commit. Sends and calendar changes bind an exact review to a short-lived token.
  • Fail closed. Ambiguous writes are never retried automatically.
  • Untrusted content. Mail and event fields are data, never agent instructions.
